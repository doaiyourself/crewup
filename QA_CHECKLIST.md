## 수동 QA 시나리오 (실계정으로 점검)

테스트 계정 4개 준비: 사장님 본인 + 점장/직원/알바용 카카오 계정(또는 지인).

### A. 라우팅·접근 제어
- [ ] 사장님 로그인 → `/admin` 진입, 미인증이면 인증 게이트
- [ ] 점장 로그인 → `/admin` 진입 (인증 게이트 통과 매장)
- [ ] 직원/알바 로그인 → `/me`로 이동
- [ ] 직원/알바가 주소창에 `/admin` 직접 입력 → `/me`로 튕겨남
- [ ] 사장님/점장이 `/me` 직접 입력 → `/admin`으로 튕겨남
- [ ] 로그아웃 상태에서 `/admin`, `/me` → `/login`

### B. 사장님(owner)
- [ ] 매장 생성 → 사업자 진위확인 통과 → active 전환
- [ ] 직원 초대(코드/링크/카카오 공유) → 합류 확인
- [ ] 점장 임명(역할 변경) 동작
- [ ] 직원 시급/직책 수정·퇴사 처리
- [ ] 출퇴근 승인/정정, 급여 계산·확정, 스케줄 편성, 계약서 발행
- [ ] 키오스크 PIN 설정 + 기기 연결 링크 생성
- [ ] 출퇴근 방식 GPS/QR 설정

### C. 점장(manager) — 권한 경계 집중 점검
- [ ] 직원·알바 초대 가능
- [ ] 직원·알바 시급/직책/퇴사 가능
- [ ] **다른 점장/사장 카드 "관리" → "사장님만 관리" 안내 뜨고 수정 막힘** (0008)
- [ ] 출퇴근 승인/급여/스케줄/계약서 가능
- [ ] (음성 테스트) 개발자도구로 본인 역할 owner 변경 시도 → 서버 거부

### D. 직원(employee) / 알바(parttimer)
- [ ] 출근/퇴근 — GPS 매장: 매장 밖이면 거리 경고, 매장 근처에서 성공
- [ ] 출근/퇴근 — QR 매장: QR 스캔 요구
- [ ] 본인 출퇴근 히스토리/통계 표시
- [ ] 본인 스케줄·급여명세서·계약서 조회 (남의 것 안 보임)
- [ ] (음성 테스트) `/api/clock` 안 거치고 attendance 직접 기록 시도 → RLS 거부 (0008)

### E. 멀티 매장
- [ ] A매장 점장 + B매장 알바인 사용자 → 상단 매장 전환 시 admin/me 화면 올바르게 전환

### F. 데이터 격리
- [ ] 점장이 다른 매장 직원 급여 URL(`/payslip?user=…`) 접근 → 차단
