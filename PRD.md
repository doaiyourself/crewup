# Crew Up — 제품 요구사항 정의서 (PRD)

> 알바 출퇴근·근로계약·급여 관리 서비스
> 작성일: 2026-06-02 · 버전: v0.1 (초안)

---

## 1. 배경 (Background)

- 알바몬이 운영하던 **크루몬(CrewMon)** 앱이 **2026년 6월 말 서비스 종료** 예정.
- 크루몬은 알바생의 **출퇴근 관리 / 근로계약서 / 급여 계산**을 제공하던 사장님용 알바 관리 도구였음.
- 대체 서비스 부재로 인해 **자체 알바 관리 서비스 "Crew Up"** 을 직접 개발한다.

## 2. 목표 (Goals)

| 구분 | 목표 |
|------|------|
| 핵심 | 크루몬의 핵심 3대 기능(출퇴근·근로계약·급여)을 동등 이상으로 대체 |
| 사용성 | 모바일 우선(Mobile-first) 반응형 웹앱으로 어디서나 사용 |
| 확장성 | 추후 **네이티브 모바일 앱(iOS/Android)** 으로 확장 가능한 구조 |
| 인증 | **카카오 SSO** 기반 간편 로그인 |
| 관리 | 강력한 **관리자(사장님/점장) 페이지** 제공 |

### 비목표 (Non-Goals, 초기 버전 제외)
- 4대보험 자동 신고 연동 (추후 검토)
- 세무/회계 시스템 연동 (추후 검토)
- 알바 채용/구인 기능 (관리에만 집중)

## 3. 사용자 & 권한 (Roles & Permissions)

4단계 역할로 구분한다.

| 역할 | 설명 | 주요 권한 |
|------|------|-----------|
| **사장님 (Owner)** | 사업주/대표. 매장(사업장)의 최고 관리자 | 매장 생성·삭제, 전 직원 관리, 급여 확정, 계약서 발행, 모든 데이터 열람, 점장 임명 |
| **점장 (Manager)** | 매장 운영 관리자 | 출퇴근 승인, 스케줄 편성, 급여 검토, 직원/알바 등록, (사장님이 부여한 범위 내) 관리 |
| **직원 (Employee)** | 정규/계약직 근로자 | 본인 출퇴근, 본인 스케줄·급여 조회, 본인 계약서 열람·서명 |
| **알바 (Part-timer)** | 시간제 근로자 | 본인 출퇴근, 본인 스케줄·급여 조회, 본인 계약서 열람·서명 |

> 권한은 **매장(사업장) 단위**로 부여된다. 한 사용자가 여러 매장에 서로 다른 역할로 소속될 수 있다. (예: A매장 사장님이면서 B매장 알바)

## 4. 핵심 기능 (Core Features)

### 4.1 출퇴근 관리 (Attendance)
- 알바/직원의 **출근(Clock-in) / 퇴근(Clock-out)** 기록
- 위치 기반 출퇴근 검증 (GPS, 매장 반경 내 체크 — 선택적)
- QR 코드 / PIN 출퇴근 방식 (위변조 방지)
- 휴게시간 자동/수동 차감
- 지각·조퇴·결근 표시
- 점장/사장님의 **출퇴근 승인·수정** (오류 정정)
- 실시간 근무 현황 대시보드 (현재 근무 중인 인원)

### 4.2 근로계약서 (Employment Contract)
- 표준근로계약서 템플릿 (고용노동부 양식 기반)
- 항목: 근로개시일, 근무장소, 업무내용, 근로시간, 휴게시간, 임금(시급/월급), 주휴수당, 4대보험 여부 등
- **전자 서명** (알바·사장 양측 서명)
- PDF 발행 및 보관
- 계약 갱신/만료 관리

### 4.3 급여 계산 (Payroll)
- 출퇴근 기록 기반 **자동 급여 산정**
- 시급 × 근무시간 + **주휴수당** 자동 계산
- 야간(22~06시)·연장·휴일 **가산수당** 계산
- 4대보험·소득세 공제(원천징수) 계산
- 급여명세서 생성 및 발송
- 급여 확정 → 정산 내역 보관
- 월별/기간별 급여 리포트

### 4.4 스케줄 관리 (Scheduling)
- 주간/월간 근무표 편성 (드래그앤드롭)
- 알바별 희망 근무시간 신청
- 스케줄 충돌·초과근무 경고
- 스케줄 변경 알림 (카카오 알림톡 / 푸시)

### 4.5 관리자 페이지 (Admin) — **중요**
- 매장(사업장) 관리: 다중 매장 지원
- 직원/알바 명단 관리 (초대·등록·퇴사 처리)
- 통계 대시보드: 인건비, 근무시간, 출근율 등 (차트)
- 알림 발송 관리
- 권한 관리

### 4.6 알림 (Notifications)
- 카카오 알림톡 (출퇴근 누락, 스케줄 변경, 급여 명세 등)
- 웹 푸시 / (추후) 앱 푸시

## 5. 기술 스택 (Tech Stack)

> "글로벌 표준 웹앱에서 가장 많이 쓰이는 스택" + "추후 모바일 앱 확장" 기준으로 선정.

### 5.1 프론트엔드 / 풀스택
- **Next.js 14+ (App Router)** + **React** + **TypeScript**
- **Tailwind CSS** + **shadcn/ui** (반응형, 모바일 우선)
- **TanStack Query** (서버 상태 관리)
- **Recharts** (관리자 통계 차트)

### 5.2 백엔드 / DB — **Supabase 올인원**
- **Supabase** 한 플랫폼으로 통합 (별도 백엔드 서버 불필요)
  - **Postgres DB**: 데이터 저장
  - **`supabase-js` 클라이언트**: 쿼리 (초기엔 별도 ORM 없이 시작, 필요 시 Drizzle 추가 가능)
  - **Storage**: 계약서 PDF 등 파일 저장
  - **Realtime**: 실시간 근무 현황 (선택적)
  - **RLS(Row Level Security)**: 매장·역할 단위 데이터 접근 제어
- Next.js Route Handlers는 보조적으로만 사용 (서버 작업, 알림 발송 등)

### 5.3 인증 — **Supabase Auth**
- **Supabase Auth**가 **카카오 OAuth를 기본 지원** → 별도 인증 라이브러리 불필요
- 역할 기반 접근 제어(RBAC): `Membership` 테이블 + **Postgres RLS 정책**으로 매장·역할별 데이터 격리
- 모바일 앱 확장 시에도 동일한 Supabase Auth 세션 공용

### 5.4 인프라 / 배포
- **Vercel** (웹 배포)
- 파일 저장(계약서 PDF 등): **Supabase Storage**
- 알림톡: 카카오 비즈메시지 (또는 솔라피 등 대행사)

### 5.5 모바일 앱 (추후 확장)
- **React Native + Expo** — 웹과 React/TypeScript 지식·일부 로직 공유
- 또는 웹앱을 **PWA**로 우선 제공 후 네이티브 전환
- 인증/API 백엔드는 웹앱과 공용

### 5.6 개발 환경
- **Claude Desktop → Code 탭** 에서 바이브 코딩으로 개발
- GitHub: `https://github.com/doaiyourself/crewup.git`
- 패키지 매니저: pnpm (권장)

## 6. 화면 구성 (주요 페이지)

### 공통
- 로그인 (카카오 SSO)
- 온보딩 (역할/매장 연결)

### 알바·직원
- 홈(오늘 일정/출퇴근 버튼)
- 출퇴근 화면
- 내 스케줄
- 내 급여 / 명세서
- 내 근로계약서

### 점장·사장님 (관리자)
- 관리 대시보드 (근무 현황, 인건비 통계)
- 직원/알바 관리
- 스케줄 편성
- 출퇴근 승인·정정
- 급여 산정·확정
- 계약서 발행·관리
- 매장 설정 / 권한 관리

## 7. 데이터 모델 (초안)

- **User** — 사용자 (카카오 연동 식별자, 이름, 연락처)
- **Store** — 매장(사업장) (상호, 주소, 사업자번호, 위치좌표)
- **Membership** — User ↔ Store 소속 (role: owner/manager/employee/parttimer, 시급/급여조건)
- **Attendance** — 출퇴근 기록 (clock-in/out, 휴게, 상태, 승인자)
- **Schedule** — 근무 스케줄
- **Contract** — 근로계약서 (내용, 서명, PDF URL, 상태)
- **Payroll** — 급여 정산 (기간, 근무시간, 수당, 공제, 실지급액, 상태)
- **Notification** — 알림 기록

## 8. 개발 로드맵 (제안)

> **개발 우선순위: 화면(UI) 먼저 → 출퇴근 MVP** — 백엔드 셋업은 뒤로 미루고, 먼저 목업(mock) 데이터로 화면을 만들어 동작을 눈으로 확인한다.

| 단계 | 범위 |
|------|------|
| **Phase 0-UI ⭐ (지금)** | **화면 우선 개발** — Next.js 프론트만, **임의의 직원 10명** 목업 + **출근/퇴근 버튼** UI 동작 |
| **Phase 0-BE** | 백엔드 셋업 (Supabase 연결 + 카카오 SSO + 역할/매장 DB 모델) — *나중에* |
| **Phase 1** | **출퇴근 MVP 완성** — 목업을 실제 DB로 교체, 출퇴근 기록 영속화 + 실시간 근무 현황 |
| **Phase 2** | 스케줄 관리 + 관리자 대시보드(통계) |
| **Phase 3** | 근로계약서 (전자서명·PDF) |
| **Phase 4** | 급여 계산·명세서 + 카카오 알림톡 |
| **Phase 5** | PWA / 네이티브 앱(React Native) 확장 |

## 9. 주요 고려사항 / 리스크

- **노동법 정합성**: 주휴수당·최저임금·가산수당 계산 로직은 법정 기준 준수 필요 (최저임금 매년 변경 반영)
- **개인정보보호**: 주민번호·계좌 등 민감정보 암호화 저장, 개인정보처리방침 필요
- **전자서명 법적 효력**: 전자문서 및 전자거래 기본법 준수
- **카카오 알림톡**: 템플릿 사전 심사, 발신프로필 등록 필요
- **데이터 마이그레이션**: 크루몬에서 기존 데이터 이관 가능 여부 확인 (6월 말 종료 전)
